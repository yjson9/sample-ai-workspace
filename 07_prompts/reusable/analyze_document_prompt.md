# 재사용 프롬프트 — 입력 문서 분석 (analyze_document)

## 목적
`01_inputs/`에 등록된 입력 문서를 분석해 요구사항·용어·리스크·다음 작업을 도출할 때 사용합니다.

## 사용 방법
분석 대상 문서(경로 또는 문서ID)를 지정해 아래 프롬프트를 AI에게 전달합니다.
분석 결과는 `03_working/analysis/`에 저장합니다.

---

## 프롬프트

```
너는 지정된 입력 문서를 분석한다.

[전제]
- 먼저 00_context/의 기준 문서(특히 AI_INSTRUCTIONS.md, OUTPUT_STANDARDS.md, SECURITY_RULES.md)를 확인한다.
- 근거는 지정된 입력 문서와 01_inputs/의 자료를 우선한다.
- 모든 판단에 신뢰도(확정/추정/확인필요/미확인)를 표기한다.
- 근거 없는 내용을 확정으로 쓰지 않는다.

[분석 대상]
- 문서: {문서 경로 또는 문서ID}

[출력 형식] — 아래 순서와 제목을 그대로 사용한다.
1. 분석 대상
   - 문서명/경로/버전
2. 문서 요약
   - 핵심 내용을 5~10줄로 요약
3. 핵심 요구사항
   - 목록 형태, 각 항목에 신뢰도 표기
4. 주요 용어
   - 용어 / 의미 / 신뢰도 / 출처 (GLOSSARY 후보)
5. 산출물 후보
   - 이 문서로부터 만들 수 있는 산출물과 형태
6. 확인 필요 사항
   - 모호하거나 사람 판단이 필요한 항목 (OPEN_QUESTIONS 후보)
7. 리스크
   - 요구사항 충돌, 범위 불명확, 보안 우려 등
8. 다음 작업
   - 권장 후속 단계
9. 생성/수정 파일 제안
   - 어떤 파일을 어디에(폴더 경로) 만들지/고칠지 제안

[주의]
- 원본 입력 문서를 수정하지 않는다.
- 개인정보·접속정보가 문서에 있으면 그대로 옮기지 말고 확인필요로 표시한다.
```
