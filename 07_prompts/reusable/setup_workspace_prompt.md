# 재사용 프롬프트 — 워크스페이스 초기화 (setup_workspace)

## 목적
새 프로젝트에서 `sample-ai-workspace`를 복사해 `ai-workspace`로 만든 뒤,
프로젝트에 맞게 기준 문서를 초기화할 때 사용합니다.

## 사용 방법
아래 프롬프트를 AI에게 그대로 전달합니다. 필요 시 프로젝트 정보를 함께 제공합니다.

---

## 프롬프트

```
너는 이 프로젝트의 AI 작업공간(ai-workspace)을 초기화한다.

[먼저 할 일]
1. 00_context/ 아래 다음 문서를 읽고 현재 상태를 파악한다:
   PROJECT_OVERVIEW.md, AI_INSTRUCTIONS.md, SECURITY_RULES.md,
   WORKFLOW.md, OUTPUT_STANDARDS.md, GLOSSARY.md, DOCUMENT_INDEX.md
2. 루트의 AGENTS.md와 README.md도 확인한다.

[프로젝트 정보 수집]
- 다음 정보를 나에게 질문하거나, 내가 제공한 자료(01_inputs/)에서 확인한다:
  프로젝트명 / 목적 / 배경 / 대상 사용자 / 주요 기능·범위 / 제외 범위 /
  기술스택 / 주요 이해관계자 / 산출물 목록 / 참고 자료.
- 확인되지 않은 항목은 임의로 채우지 말고 `{작성 필요}`로 남기거나
  06_decisions/OPEN_QUESTIONS.md에 질문으로 기록한다.

[해야 할 작업]
1. PROJECT_OVERVIEW.md의 {작성 필요}를 수집한 정보로 채운다.
   - 확정된 정보만 확정 표현으로 쓰고, 불확실한 것은 `추정`/`확인필요`로 표기한다.
2. DOCUMENT_INDEX.md를 초기화한다.
   - 예시 행을 지우고, 01_inputs/에 등록된 실제 입력자료를 색인에 추가한다.
   - 아직 자료가 없으면 빈 표(헤더만) 상태로 둔다.
3. AI_INSTRUCTIONS.md를 프로젝트 성격에 맞게 보완한다.
   - 기술스택 특성, 산출물 형태, 보안 제약을 반영한다.
   - 공통 규칙(AGENTS.md)과 충돌하지 않게 한다.
4. 기술스택 / 산출물 / 보안 제약을 각 기준 문서에 반영한다.

[하지 말 것]
- 불필요한 도메인 특화 내용을 새로 만들지 않는다(범용성 유지).
- 개인정보·접속정보·실데이터·비밀값을 어떤 문서에도 넣지 않는다.
- 근거 없는 내용을 확정으로 쓰지 않는다.

[마무리 보고]
- 채운 항목 / 남은 `{작성 필요}` 항목 / OPEN_QUESTIONS에 추가한 질문을 요약한다.
- 다음에 필요한 작업(입력자료 등록, 첫 요청서 작성 등)을 제안한다.
```

## 참고
- 이 프롬프트 실행 후, 실제 입력자료를 `01_inputs/`에 넣고 다시 `DOCUMENT_INDEX.md`를 갱신하세요.
- 사용 이력은 `07_prompts/history/`에 남기는 것을 권장합니다.
