# DOCUMENT_INDEX — 문서 색인

입력 문서와 산출물을 색인합니다. 새 문서를 등록하거나 상태가 바뀌면 이 표를 갱신합니다.

- `상태` (대상에 맞는 체계 사용 — `OUTPUT_STANDARDS.md` §5):
  - 입력자료(`01_inputs/`): `received` / `verified` / `superseded` / `archived`
  - 작업·산출물(`03_working/`,`04_outputs/`): `draft` / `review_requested` / `approved` / `rejected` / `archived`
- `문서유형`: 표준 / 요구사항 / 참고 / 샘플 / 원본 / 분석 / 산출물 등

## 색인 (실제 등록 문서)

> 이 표에는 **실제 문서만** 등록합니다. 형식 예시는 아래 "작성 예시" 블록을 참고하세요.

| 문서ID | 문서명 | 위치 | 문서유형 | 작성자/출처 | 등록일 | 버전 | 상태 | 요약 | 관련 요청ID |
|--------|--------|------|----------|-------------|--------|------|------|------|-------------|

## 작성 예시 (참고용 — 실제 데이터 아님, 색인 표에 복사하지 말 것)

```text
| 문서ID  | 문서명        | 위치                          | 문서유형 | 작성자/출처 | 등록일     | 버전 | 상태  | 요약            | 관련 요청ID |
| DOC-000 | 프로젝트 개요 | 00_context/PROJECT_OVERVIEW.md | 산출물   | 내부        | 2026-07-14 | v1  | draft | 목적·범위 정의  | -           |
```

---

## 작성 지침

- **문서ID**: `DOC-###` 형식으로 부여한다.
- **위치**: 워크스페이스 루트 기준 상대 경로.
- **버전**: `v1`, `v2` … 로 표기.
- **관련 요청ID**: 해당 문서와 연결된 `02_requests/`의 요청ID(`REQ-###`).
- 원본(`01_inputs/raw/`)은 위치·버전을 반드시 기록해 추적 가능하게 한다.
