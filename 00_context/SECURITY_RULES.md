# SECURITY_RULES — 보안 규칙

이 워크스페이스에서 작업하는 사람과 AI 모두 아래 규칙을 지킨다.
규칙 위반 소지가 있으면 작업을 멈추고 `확인필요`로 표시한다.

---

## 1. 저장 금지 항목

다음 항목은 이 워크스페이스 어디에도 저장하지 않는다.

- **개인정보**: 실명, 주민등록번호/식별번호, 전화번호, 이메일, 주소 등
- **실데이터**: 운영 환경의 실제 고객/사용자 데이터
- **계정 정보**: 아이디, 비밀번호
- **비밀값**: API Key, 토큰, 인증서, 비밀키
- **접속 정보**: 내부 IP, 서버 호스트명, 포트, 접속 URL, DB 접속 문자열

---

## 2. 원본 보호

- 원본 문서·원본 소스코드를 **임의로 수정·삭제하지 않는다.**
- 가공이 필요하면 원본을 복사해 `03_working/`에서 작업한다.
- 원본의 위치·버전은 `00_context/DOCUMENT_INDEX.md`에 기록한다.

---

## 3. 예시 데이터 규칙

- 문서·코드의 예시는 **가상 값**만 사용한다.
- 예시 이메일은 `user@example.com`, 이름은 `홍길동`/`Jane Doe` 같은 가상값을 사용한다.
- 실제 값에서 일부만 가린 형태(마스킹된 실데이터)도 사용하지 않는다.

---

## 4. Git 커밋 규칙

- 민감 자료는 Git에 커밋하지 않는다.
- `.gitignore`에 비밀값·로그·임시파일 패턴이 기본 포함되어 있다.
- 보안 수준이 높은 프로젝트는 `01_inputs/raw/` 등을 `.gitignore`에 **추가**한다.
- 실수로 커밋했다면 즉시 담당자에게 알리고 이력에서 제거를 요청한다.

---

## 5. AI 산출물의 지위

- **AI가 만든 모든 산출물은 사람이 검토하기 전까지 초안(draft)이다.**
- 확정되지 않은 내용을 확정 사실처럼 외부에 공유하지 않는다.
- 산출물에는 `확정`/`추정`/`확인필요`/`미확인` 신뢰도를 표기한다.

---

## 6. 위반 발견 시

- 위 규칙 위반이 의심되면 작업을 중단한다.
- 해당 내용을 `05_feedback/review_requests/` 또는 담당자에게 보고한다.
- 이미 저장된 민감 자료는 즉시 제거를 요청한다.
