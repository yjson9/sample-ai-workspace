# CHANGE_LOG — 변경 이력

문서·구조·주요 산출물의 변경 이력을 기록합니다.

> **운영 정책 (기록 시작 시점):**
> 이 CHANGE_LOG는 **이 워크스페이스를 새 프로젝트로 복사한 시점부터** 운영합니다.
> 즉, `sample-ai-workspace` 템플릿 자체를 개발/보완하는 변경(가이드·스크립트·템플릿 추가 등)은
> 여기에 기록하지 않습니다(그 이력은 Git 커밋 이력으로 관리).
> 초기화(`setup_workspace_prompt.md`) 후, 실제 프로젝트 운영 중의 변경부터 아래 표에 남기세요.

> 이 표에는 **실제 변경만** 등록합니다. 형식은 아래 "작성 예시" 블록을 참고하세요.
> (아직 등록된 변경이 없으면 헤더만 유지합니다.)

| 날짜 | 변경 대상 | 변경 내용 | 변경 사유 | 영향 범위 | 작성자 |
|------|-----------|-----------|-----------|-----------|--------|

## 작성 예시 (참고용 — 실제 데이터 아님)

```text
| 2026-07-14 | 워크스페이스 구조 | 초기 폴더 구조 생성 | 프로젝트 시작 | 전체 | 작성자 |
```

---

## 작성 지침

- 되돌리기 어려운 변경(대량 수정/삭제/구조 변경)은 반드시 기록한다.
- `변경 대상`은 문서ID 또는 경로로 명시한다.
- 결정에 근거한 변경이면 `DECISION_LOG.md`의 `DEC-###`를 함께 적는다.
